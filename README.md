
## Определения

__Командная оболочка (shell)__ - это программа, которая принимает команды, введенные с клавиатуры, и передает их операционной системе для выполнения.  


__Эмулятор терминала__ - он предоставляет доступ к командной оболочке (отвечает за ввод и вывод текста)  

__Приглашение к вводу (shell prompt)__ - появляется когда, командная оболочка готова принять ввод. Если в конце вместо $ расположен #, то это означает, что сеанс в терминале обладает привилениями суперпользователя.

__Домашняя директория__ - Это каталог, в котором пользователь может осуществлять запись в файлы с правами обычного пользователя. Этим каталогом является `/home/user` или сокращённая форма `~`.  

__Относительный путь__ - начинается в рабочем каталоге и для обозначения относительных позиций в дереве файловой системы используется пара специальных символов: `.` (точка) и `..` (точка-точка). Символ `.`(точка) обозначает рабочий каталог, а символ `..` (точка-точка) обозначает каталог, родительский по отношению к рабочему.   

__Абсолютный путь__ - Абсолютный путь начинается с корневого каталога `/` и перечисляет ветви дерева например '/usr/bin', отделяющие корень от желаемого каталога или файла.  

__Путь (pathname)__ - это маршрут, перечисляющий ветви дерева, по которым нужно пройти, чтобы достигнуть желаемого каталога, например абсолютный `/home/user/Documents` или относительный `./projects`или без точки `projects/` который будет искаться из текущей рабочей директории.  

__Текущий рабочий каталог__ - это каталог, в котором находится разработчик (если сделать cd ~/.config то именно он будет текущим рабочим каталогом)

## Навигация  

### pwd  
__pwd__ - расшифровывается как  __print working directory__ и выводит название текущего рабочего каталога `/mnt/d/Learn`  



### ls  
__ls__ - выводит список файлов и каталогов в текущей или указанной директории  
> __-a__ / __--all__ - Список всех (all) файлов, даже с именами, начинающимися с точки, которые обычно не выводятся (то есть скрытых)  
> __-A__ / __--almost-all__ - Действует подобно параметру -a, но не выводит каталоги . (текущий рабочий каталог) и .. (родительский каталог)  
> __-d__ / __--directory__ - Обычно в присутствии этого параметра команда ls выводит информацию о самом каталоге, а не его содержимое. Используйте этот параметр в сочетании с параметром -l, чтобы получить дополнительную информацию о каталоге, а не о его содержимом  
**ЗАМЕЧАНИЕ ПО ОПРЕДЕЛЕНИЮ ИЗ КНИГИ** — обычно `ls`, встретив каталог, показывает его содержимое (файлы внутри). С флагом `-d` `ls` вместо этого выводит саму запись каталога (его имя/путь) — так, как будто это обычный файл, не заглядывая внутрь.


> __-F__ / __--classify__ - Добавляет в конец каждого имени символ-индикатор (например, прямой слеш, если это имя каталога)  
> __-h__ / __--human-readable__ - При использовании длинного формата вывода отображает размеры файлов не в байтах, а в величинах с единицами измерения  
> __-l__ - Выводит результаты с использованием длинного формата  
> __-r__ / __--reverse__ - Выводит результаты в обратном порядке. Обычно команда ls выводит результаты в алфавитном порядке  
> __-S__  - Сортировать результаты по размеру (size)  
> __-t__ - Сортировать результаты по времени (time) последнего изменения  

> __-i__ - Извлекает (отображает) информацию об `inode / индексном узле`

> __Поля вывода в ls__ - например `-rw-r--r--    1   root    root    35762961    2017-04-03  11:05   example.py`  
> __-__ - Первый символ указывает тип файла. Например, символом дефиса `-` обозначаются обычные файлы, а символом `d` — каталоги.  
> __rw-__ - Следующие три символа сообщают о правах доступа для владельца файла  
> __r--__ - Cледующие три — для членов группы, которой принадлежит файл  
> __r--__ - Последние три — для всех остальных  
> __1__ - Число жестких ссылок на файл  
> __root__ - Имя пользователя, владеющего файлом  
> __root__ - Имя группы, владеющей файлом  
> __35762961__ - Размер файла в байтах  
> __2017-04-03  11:05__ - Дата и время последнего изменения файла  
> __example.py__ - Имя файла  


### cd   
__cd__ - расшифровывается как __change directory__ и позволяет переместиться в другую директорию. Аргументом выступает `pathname` который может быть относительный или абсолютный! Если, вначале стоит `/`, то путь считается абсолютным `/usr/bin/`.  

> __Каталоги в системе Linux__  
> __/__ - Корневой каталог, откуда все начинается  
> __/bin__ - Содержит двоичные (binaries) файлы (программы), необходимые для загрузки и функционирования системы  
> __/boot__ - Содержит ядро Linux, образ начального RAM-диска (с драйверами, необходимыми на этапе загрузки) и сам загрузчик.  
> __/dev__ - Специальный каталог, содержащий узлы устройств. «Все сущее есть файл» применяется также к устройствам. Здесь ядро хранит список всех известных ему устройств  
> __/etc__ - Каталог /etc содержит все системные конфигурационные файлы. Здесь же хранится коллекция сценариев командной оболочки, запускающих системные службы во время загрузки.   
> __/home__ - В обычных конфигурациях каждому пользователю выделяется свой домашний каталог в каталоге /home. Простые пользователи могут записывать что-нибудь только в файлы, находящиеся в их домашних каталогах. Это ограничение защищает систему от необдуманных действий пользователей  
> __/lib__ - Содержит файлы разделяемых библиотек, используемых основными системными программами. Они напоминают библиотеки DLL в Windows  
> __/lost+found__ - Каждый раздел или устройство, отформатированные с использованием файловой системы Linux, такой как ext3, будут иметь этот каталог. Он используется на случай частичного восстановления повреждений в файловой системе.  
> __/media__ - В современных системах Linux каталог /media будет содержать точки монтирования съемных носителей, таких как USB-диски, CD-ROM и т. д., которые монтируются в момент подключения  
> __/mnt__ - В старых системах Linux каталог /mnt содержал точки монтирования съемных носителей, монтируемых вручную  
> __/opt__ - Каталог /opt используется для установки «необязательного» (optional) программного обеспечения, в основном для установки коммерческого программного обеспечения  
> __/proc__ - Специальный каталог. Не является фактической файловой системой, в том смысле, что файлы в этом каталоге не хранятся на жестком диске. Это виртуальная файловая система, поддерживаемая ядром Linux. Файлы в ней являются «глазками», через которые можно заглянуть в ядро. Эти файлы доступны для чтения и помогают «увидеть» компьютер глазами ядра  
> __/root__ -  Домашний каталог пользователя root  
> __/sbin__ - Каталог содержит системные двоичные файлы (system binaries). Эти программы выполняют жизненно важные задачи и обычно запускаются только суперпользователем  
> __/tmp__ - Каталог /tmp играет роль временного хранилища для временных файлов, создаваемых разными программами. В некоторых конфигурациях этот каталог принудительно очищается при каждой перезагрузке системы  
> __/usr__ - Дерево каталогов /usr является, пожалуй, самым объемным в системе Linux. В нем хранятся все программы и файлы поддержки, используемые обычными пользователями  
> __/usr/bin__ - Каталог /usr/bin содержит выполняемые программы, установленные дистрибутивом Linux. Очень часто в этом каталоге хранятся тысячи программ  
> __/usr/lib__ - Содержит разделяемые библиотеки для программ в /usr/bin  
> __/usr/local__ -  Дерево каталогов /usr/local используется для установки тех программ, которые не входят в состав дистрибутива, но должны быть доступны всем пользователям в системе. Программы, собираемые из исходных текстов, обычно устанавливаются в /usr/local/bin.  
> __/usr/sbin__ - Содержит дополнительные программы для администрирования  
> __/usr/share__ - Каталог /usr/share содержит все разделяемые данные, используемые программами в /usr/bin, в том числе конфигурационные файлы с настройками по умолчанию, ярлыки, фоновые изображения для рабочего стола, звуковые файлы и т. д  
> __/usr/share/doc__ - Большинство пакетов, установленных в системе, содержат документацию. Вся эта документация, организованная по пакетам, хранится в каталоге /usr/share/doc  
> __/var__ - За исключением /tmp и /home, все упоминавшиеся выше каталоги остаются относительно статичными; то есть их содержимое почти не меняется. Дерево каталогов /var — как раз то место, где хранятся часто изменяемые данные: различные базы данных, буферные файлы, почта пользователей и пр.  
> __/bar/log__ - Каталог /var/log содержит файлы журналов с записями о различных действиях, выполнявшихся в системе. Они очень важны и должны проверяться время от времени. Наиболее полезными являются файлы /var/ log/messages и /var/log/syslog. Обратите внимание, что из соображений безопасности некоторые системы требуют привилегий суперпользователя для просмотра файлов журналов

## Операции с файлами и коталагами  
__символы подстановки (globbing) / гропповые символы (wildcards)__ - позволяют выбирать имена файлов по шаблону.  

> __групровые символы__ - Это не регулярные выражения, а отдельный язык шгаблонов, который гораздо проще!  
> __*__ - Любая последовательность любых символов  
> __?__ - Любй один символ  
> __[символы]__ - Любой один символ из указанного множества символов  
> __[!символы]__ - Любой один символ, не принадлежащий указанному множеству символов  
> __[[:класс:]]__ - Любой один символ, принадлежащий указанному классу  

> __Класс символов__ - наиболее часто используемые классы символов  
> __[:alnum:]__ - Любой алфавитно-цифровой символ  
> __[:alpha:]__ - Любой алфавитный символ  
> __[:digit:]__ - Любой цифровой символ  
> __[:lower:]__ - Любая буква в нижнем регистре  
> __[:upper:]__ - Любая буква в верхнем регистре  

> __Примеры использования групповых символов__  
> __*__ - Все имена файлов  
> __g*__ - Все имена файлов, начинающиеся с символа `g`  
> __b*.txt__ - Все имена файлов, начинающиеся с символа «b», за которым следует любое число других символов, и заканчиваю щиеся на «.txt»  
> __Data???__ - Все имена файлов, начинающиеся с символов «Data», за которыми следуют ровно три любых символа  
> __[abc]*__ - Все имена файлов, начинающиеся с символа «a», «b» или «c»  
> __BACKUP.[0-9][0-9][0-9]__ - Все имена файлов, начинающиеся с символов «BACKUP.», за которыми следуют ровно три цифровых символа  
> __[[:upper:]]*__ - Все имена файлов, начинающиеся с буквы в верхнем  регистре  
> __[![:digit:]]*__ - Все имена файлов, не начинающиеся с цифры  
> __*[[:lower:]123]__ - Все имена файлов, заканчивающиеся буквой в нижнем регистре или цифрой «1», «2» или «3»  


### cp  
__cp__ - Предназначен для копирования файлов и каталогов, однако, изначально без дополнительных флагов, может копировать только файлы, например `cp main.py copy.py`.   

> __Параметры команды cp__  
> __-a__ / __--archive__ - Скопировать файлы и каталоги со всеми атрибутами, включая идентификаторы владельцев и права доступа. Без этого параметра копии обычно получают значения атрибутов по умолчанию, определенных для пользователя, выполняющего копирование.  
> __-i__ / __--interactive__ - Запрашивать у пользователя подтверждение перед перезаписью существующего файла. Если этот параметр отсутствует,  команда cp просто перезапишет существующие файлы (без предупреждения)  
> __-r__ / __--recursive__ - Рекурсивно копировать каталоги и их содержимое. Это обязательный параметр (или параметр -a) при копировании каталогов  
> __-u__ / __--update__ - При копировании файлов из одного каталога в другой копировать только файлы, отсутствующие в каталоге назначения или более новые. Этот параметр удобно использовать при копировании большого числа файлов, так как он позволяет пропустить файлы, которые не требуется копировать  
> __-v__ / __--verbose__ - Выводить информационные сообщения в процессе копирования  

> __Примеры использования команы cp__  
> __cp file1 file2__ - Скопирует file1 в file2. Если file2 существует, он будет затерт новым файлом file1. Если file2 отсутствует, он будет создан  
> __cp -i file1 file2__ - То же, что и выше, но если файл file2 существует, у пользователя будет запрошено подтверждение перед перезаписью файла  
> __cp file1 file dir1__ - Скопирует file1 и file2 в каталог dir1. Каталог dir1 должен существовать  
> __cp dir1/* dir2__ - С использованием группового символа. Скопирует все файлы из каталога dir1 в каталог dir2. Каталог dir2 должен существовать  
> __cp -r dir1 dir2__ - Скопирует содержимое каталога dir1 в каталог dir2. Если каталог dir2 не существует, он будет создан и заполнен содержимым каталога dir1. Если каталог dir2 существует, тогда в него будет скопирован сам каталог dir1 (вместе с его содержимым)  



### mv  
__mv__ - выполняет операции перемещения и переименования файлов в зависимости от особенностей использования. В любом случае исходный файл исчезает после операции.   

> __Папаметры команды mv__  
> __-i__ / __--interactive__ - Запрашивать у пользователя подтверждение перед перезаписью существующего файла. Если этот параметр отсутствует, коман да mv просто перезапишет существующие файлы  
> __-u__ / __--update__ - При перемещении файлов из одного каталога в другой перемещать только файлы, отсутствующие в каталоге назначения или более новые  
> __-v__ / __--verbose_- - Выводить информационные сообщения в процессе перемещения  

> __Примеры использования команы mv__  
> __mv file1 file2__ - Переместит file1 в file2. Если file2 существует, он будет заменен на новый файл file1. Если file2 отсутствует, он будет создан. В любом случае файл file1 исчезнет  
> __mv -i file1 file2__ - То же, что и выше, но если файл file2 существует, у пользователя будет запрошено подтверждение перед перезаписью файла  
> __mv file1 file2 dir1__ - Переместит file1 и file2 в каталог dir1. Каталог dir1 должен существовать  
> __mv dir1 dir2__ - Если каталог dir2 отсутствует, команда mv создаст его и переместит содержимое каталога dir1 в каталог dir2, после чего удалит dir1. Если каталог dir2 существует, команда mv переместит каталог dir1 (вместе с его содержимым) в каталог dir2  





### mkdir  
__mkdir__ - используется для создания каталогов, например `mkdir dir1`, или даже создавать больше однйо папки `mkdir dir1 dir2 dir3`  


### rm  
__rm__ - используется для удаления (remove) файлов и каталогов  

> __Параметры команды rm__  
> __-i__ / __--interactive__ -  Запрашивать у пользователя подтверждение перед удалением существующего файла. Если этот параметр отсутствует, команда rm просто удалит существующие файлы  
> __-r__ / __--recursive__ - Рекурсивно удалить каталоги. То есть вместе с каталогом будут удалены все его подкаталоги. Это обязательный параметр при  удалении каталогов  
> __-f__ / __--force__ - Игнорировать отсутствующие файлы и не запрашивать подтверждения. Этот параметр отменяет действие параметра --interactive  
> __-v__ / __--verbose__ -  Выводить информационные сообщения в процессе удаления  


> __Примеры использования команды rm__
> __rm file1__ - Просто удалит файл file1
> __rm -i file1__ - Перед удалением file1 запросит подтверждение у пользователя
> __rm -r file1 dir1__ - Удалит файл file1 и каталог dir1 со всем его содержимым
> __rm -rf file1 dir1__ - То же, что и выше, но в отсутствие file1 и/или dir1 просто продолжит работу, не выводя никаких сообщений


### ln
Жёсткие ссылки — это первоначальный способ создания ссылок в Unix. По умолчанию 
каждый файл имеет одну жёсткую ссылку, определяющую его имя. Создавая жёсткую 
ссылку, мы создаём дополнительную запись в каталоге для файла. Жёсткая ссылка не 
может указывать на файл за пределами собственной файловой системы (нельзя ссылаться 
на файл в другом разделе диска).

Пользователь не может создавать дополнительные жёсткие ссылки на каталог через `ln`.
Важно понимать: в Linux всё является файлом, и каталог — не исключение. У каталогов
ЕСТЬ жёсткие ссылки, просто они создаются автоматически самой файловой системой:
`.` — ссылка на сам каталог, `..` — ссылка на родительский. Поэтому у пустого 
каталога link count равен 2, а не 1:

>
>  `$ mkdir testdir`
>  `$ ls -lid testdir`
>  `14498 drwxr-xr-x 2 warcry warcry 4096 Jul 2 06:57 testdir`

Создать ДОПОЛНИТЕЛЬНУЮ жёсткую ссылку на каталог нельзя даже от root — на 
современном Linux это запрещено на уровне ядра (системный вызов link() возвращает 
EPERM), независимо от прав пользователя. Флаг `ln -d`/`--directory` существует в 
GNU coreutils как наследие старых Unix-систем (например, Version 7 Unix), где root 
действительно мог это делать, но в Linux эта возможность полностью закрыта — из-за 
риска создать циклы в дереве файловой системы, которые сломали бы такие утилиты, 
как find, du, fsck.

### wc
__wc__ - (word count - счётчик слов) используется для подсчета числа строк, слов и байтов в файлах. Выводит число символов перевода строки, слов и байтов в каждом указанном файле, например: `wc README.md` и даст вывод `      46     269    3170 README.md`.

> __Флаги__
> __-l__ - ограничивает вывод информации только числом строк, например: `ls /bin /usr/bin | sort | uniq | wc -l`! Может использоваться для подсчёта строк или элементов!


### head
__head__ - выводит первые строки из файла; Удобно для быстрого предпросмотра файла! По умолчанию выводит gthdst 10 строк, но флаг `-n` может установить количество строк для выаодп, например: `head ls-out.txt` или с флагом `head -n 5 ls-out.txt`!

> __Флаги__
> __-n__ - устанавливает число строк, которые надо вынести! По умолчанию, выводится первые 10 строк!

### tail
__tail__ - выводит последние строки из файла! Удобно для быстрого предпросмотра файла! По умолчанию выводит последние 10 строк, но флаг `-n` может установить количество строк для выаодп, например: `tail ls-out.txt` или с флагом `tail -n 5 ls-out.txt`! Так же, может быть установылен флаг `-f` который позволяет в режиме реальново времени следить за изменениями файлов, например файла с логами, и мгновенно при добавлении новых строк, выводит их, например: `tail -f /var/logs/messages`!  

> __Флаги__
> __-n__ - устанавливает число строк, которые надо вынести! По умолчанию, выводится последние 10 строк!
> __-f__ - позволяет в режиме реального времени следить за изменениями файлов, например файла с логами, и мгновенно при добавлении новых строк их выводит! 


### tee
__tee__ - читает данные со стандартного ввода и записывает в стандартный вывод и в файлы, например: `ls /usr/bin | tee ls.txt | grep zip`! По сути, эта утилита получает на вход текст по пайплайну и отдаёт обратно в __stdout__ а так же, попутно даёт возможность сделать запись в один или несколько файлов! На мой взгляд, это что то похожее на `>` оператор перенаправления по функционалу, только улучшеное! Да, это не его замена и не аналог, а скоре просто другой иснтрумент с схожими возможностями, и имеет различия! Он, похож на тройник для врезки в трубу, когда `>` как полное перенаправление только в одну сторону (как угол врезанный в трубу)!



### >
__>__ - оператор перенаправления стандартного вывода! Чтобы перенаправить стандартный вывод в другой файл вместо экрана, нужно добавить в команду оператор перенаправления `>` и имя файла, например `ls -laF > ls-output.txt`. Это может создать новый файл, или перезаписать существующий, так что если в файле что то было, то оно полностью перезапишется!

### >>
__>>__ - так же является оператором перенаправления стандартного вывода, НО уже добавляет вывод в конец файла, а не перезаписывает его, например: `ls -liaF >> ls-output.txt`. Оператор >> просто добавит результаты в конец файла. Если файл не существует, он будет создан.

### 2> 
__2>__ - По сути, это всё ещё оператор перенаправления вывода, но с дескриптором `2` для файлового дескриптора вывода ошибок `stder`! Применяется так же как и `>`, и действует так же, только перенаправляет стандартный поток вывода ошибок, например: `ls -liaF /bin/usr 2> ls-error.txt`! Создаёт файл если не создан, и перезаписывает его! Если в файле есть данные, то они будут утеряны (перезаписаны)!

### 2>>
__2>>__ - Действует так же как и `>>`, только соответственно перенаправляет стандартный поток вывода ошибок, например: `ls -liaF /bin/usr 2>> ls-error.txt`. Создаст файл если не существует, и добавит в данные в конец документа не трогая исходные данные!


### 2>&1
__2>&1__ - Для перенаправления двух потоков (и стандартного вывода, и вывода ошибок), используется уловка, при которой, в два подхода, сначала перенаправляется стандартный вывод __stdout__ в файл `> ls-output.txt`, а затем, перенаправляется __stderr__ поток ошибок, в __stdout__ стандартный вывод `2>&1`! Получается, что сначала мы перенаправляем __stdout__ в `ls-output.txt`, а затем __stderr__ в __stdout__ (который уже перенаправлен в файл), и таким образом, оба вывода будут перенаправлены в один файл! Перенаправление стандартного вывода ошибок всегда должно производиться после перенаправления стандартного вывода, иначе этот трюк не сработает. 

### &>
__&>__ - Это современная альтернатива`2>&1`, и работает так же, только с отдельным, специальным оператором __&>__! Он создаст файл если не создан, и перезапишет контент файла!

### &>>
__&>>__ - Этот оператор так же перенаправит оба потока, __stdout__ и __stderr__ в один файл, но уже не будет перезаписывать данные, а добавит данные в конец файла. Если файла нет, то создаст!

### 2> /dev/null
__2> /dev/null__ - Иногда вывод команды нужно отбросить. В особенности это касается служебных сообщений и сообщений об ошибках. Си
стема дает такую возможность, предоставляя специальный файл /dev/null, куда можно перенаправить вывод. Этот файл представляет системное устройство, называемое битоприемником (bit bucket) или мусорной корзиной, которое принимает любой ввод и ничего с ним не делает. Пример: `ls -liaF /bin/usr 2> /dev/null`


### |
__|__ -  оператор конвейера `|`, который позволяет стандартный вывод одной команды можно связать со стандартным вводом другой, например: `ls -liaF /usr/bin | less`.

### Разница | и >
Оператор перенаправления `>` связывает команду с файлом, а оператор конвейера `|` связывает вывод одной команды с вводом другой. Например, если от имени суперпользователя, проделать `cd /usr/bin` а затем `ls > less`, то оператор перенаправления, возьмёт вывод __stdout__ от `ls`, создаст файл `less` (а так как файл уже существует - это бинарник утилиты), то он без всяких вопросов и предупреждений его затрёт, и впишет в него текст вывода из `ls`! Фактически, программа будет затёрта! Оператор перенаправления без лишних предупреждений создает или затирает файлы.

> 2007-01  2007-07  2008-01  2008-07  2009-01  2009-07

- Перечисления через точки `..`
- Не допускаются пробелы в скобках (и похоже, даже вне скобок не допускается, обрубает вторую часть выражения, по этому в примере вместо пробела, разделитель `-`)

### Подстановка параметров
__$USER__ - например, переменная с именем `USER`, которая хранит имя пользователя! Пример использования: `echo $USER`, вернёт имя текущего пользователя, например `root`! 

> __printenv__ - метод для вывода всех зарегистрированных переменных (глобальных)!



### Подстановка команд $()
Подстановка команд позволяет использовать вывод команд в качестве аргументов других команд, например: `echo $(ls)`. В данном случае, вывод команды `ls` будет подставлен в качестве аргумента в команду `echo`. Если `ls` выводит `Projects  snap  usr`, то команда `echo $(ls)` выведет так же `Projects  snap  usr`!

Ещё пример: если команда `which cp` выводит `/usr/bin/cp`, то результатом команды `ls -l $(which cp)` будет `lrwxrwxrwx 1 root root 5 Mar 30 09:50 /usr/bin/cp -> gnucp`! То есть `/usr/bin/cp` подставляется как аргумент — путь к файлу (в данном случае это симлинк).

Или посложнее: `file $(ls -d /usr/bin/* | grep zip)`. В этом примере в команду `file` передаются в качестве аргумента строки, полученные от `grep`, который фильтрует по наличию в имени текста `zip`. Эти строки, в свою очередь, поступили из вывода `ls`. Команда `ls -d /usr/bin/*` выводит список всех путей внутри `/usr/bin` (это в основном файлы, а не директории), не разворачивая содержимое возможных подкаталогов — флаг `-d` тут гарантирует, что если бы среди них были папки, `ls` не стала бы заглядывать внутрь них. В результате `file` получает список отфильтрованных путей и определяет тип каждого файла (бинарник, скрипт, симлинк и т.д.).



### Альтернативный синтаксис подстановки команд - ``

Существует альтернативный синтаксис, который унаследован от более старых командных оболочек и также поддерживается `bash`. В нём вместо `$()` используются **обратные кавычки** ` `` `.

Хотя в книге сказано, что это устаревшая механика, но на мой взгляд это намного удобнее, чем знак доллара и кавычки! Я вечно путаю знак доллара с другими клавишами, и это приносит дискомфорт!

**ОДНАКО** есть очень важный момент! Обратные кавычки сложнее вкладывать друг в друга. Если нужно подставить одну команду внутрь другой, например `echo $(echo $(whoami))`, то в случае с обратными кавычками пришлось бы писать так:

```` echo `echo \`whoami\`` ````

— это труднее использовать и понимать.

Вот пример: ``ls -l `which cp` `` и он работает так же, как и `ls -l $(which cp)`!



### Экранирование

#### Двойные ковычки
__" "__ - Например `ls -l "Два слова.txt"` сработает, так как ковычки не дадут разбить название файла на две части по пробелу!

> __Исключения__ - не экранируются двойными кавычками:
> __$ (знак доллара)__ - 
> __\ (обратный слеш)__ - 
> __` (обратный апостроф)__ - 

#### Одинарные ковычки
__' '__ - в отличии от двойных ковычек, одинарные подавляют все подстановки. 
Пример: `echo 'text ~/*.txt {a,b} $(echo foo) $((2+2)) $USER'`


#### Экранирование символов ( \ )

Обратный слеш `\` перед символом отменяет его специальное значение для командной оболочки — экранирует **только один** символ (в отличие от кавычек, которые экранируют всё внутри себя).

**Пример — внутри двойных кавычек:**

```bash
echo "Баланс счета пользователя $USER: \$5.00"
```
Вывод: `Баланс счета пользователя me: $5.00`

Здесь `$USER` подставился как обычно, а `\$` — нет, потому что `\` отменил специальное значение `$` именно в этом месте.

**Пример — в именах файлов:**

Символы вроде `$`, `!`, `&`, пробел и др. имеют особое значение для шелла. Если такой символ нужен прямо в имени файла — его экранируют слешем:

```bash
mv bad\&filename good_filename
```

**Важно:**
- Чтобы вставить сам символ `\`, его тоже нужно экранировать: `\\`.
- Внутри **одиночных** кавычек `'...'` обратный слеш теряет свою специальную роль и воспринимается как обычный символ (не экранирует ничего).

### Управляющие последовательности (control codes)

Обратный слеш используется не только как экранирующий символ, но и как часть **управляющих последовательностей** — обозначений специальных кодов ASCII (изначально использовались для команд устройствам вроде телетайпа).

Основные управляющие последовательности:

- __\a__ — звонок (звуковой сигнал)
- __\b__ — забой (backspace)
- __\n__ — новая строка (перевод строки)
- __\r__ — возврат каретки
- __\t__ — табуляция

Идея пришла из языка C и была заимствована многими языками, включая командную оболочку.

**Как включить интерпретацию:**
- Параметр `-e` у команды `echo`, например: `echo -e "Привет\nПока"`
- Или конструкция `$' '`, например: `echo $'Привет\nПока'`


## Работа с клавиатурой


### Редактирование командной строки (Readline)

Bash использует библиотеку **Readline** для редактирования командной строки. Кроме стрелок влево/вправо, есть и другие полезные комбинации (учить все необязательно — выбирайте удобные).

Комбинации с ALT иногда перехватываются графическим интерфейсом, но в виртуальной консоли работают все.

**Перемещение курсора:**

- __CTRL+A__ — в начало строки
- __CTRL+E__ — в конец строки
- __CTRL+F__ — на символ вперёд (как →)
- __CTRL+B__ — на символ назад (как ←)
- __ALT+F__ — на слово вперёд
- __ALT+B__ — на слово назад
- __CTRL+L__ — очистить экран (то же, что `clear`)


### Клавиша META и проблема с ALT в Windows Terminal

В документации Readline (`man bash`, раздел READLINE) встречается термин **клавиша meta**. На современных клавиатурах её роль играет **ALT**. Если ALT не срабатывает — можно нажать и отпустить **ESC**, эффект будет тот же (это работает и в WSL/Windows Terminal, где ALT часто перехватывается системой).

**Редактирование текста:**

- __CTRL+D__ — удалить символ в позиции курсора
- __CTRL+T__ — поменять местами два символа (текущий и предыдущий)
- __ALT+T__ — поменять местами два слова (текущее и предыдущее)
- __ALT+L__ — сделать строчными буквы от курсора до конца слова
- __ALT+U__ — сделать заглавными буквы от курсора до конца слова

**У меня в WSL / Windows Terminal комбинации с ALT не работают** — вместо `ALT+KEY` нужно нажать **ESC**, отпустить, затем отдельно нажать нужную клавишу (например, `ESC` → `T`).


### Вырезание и вставка (killing and yanking)

В Readline это называется **killing and yanking** — обычное вырезание/вставка. Вырезанный текст попадает в кольцевой буфер **kill-ring**.

- __CTRL+K__ — удалить от курсора до конца строки
- __CTRL+U__ — удалить от курсора до начала строки
- __ALT+D__ — удалить от курсора до конца текущего слова
- __ALT+BACKSPACE__ — удалить от курсора до начала слова (если курсор уже в начале — удаляется предыдущее слово)
- __CTRL+Y__ — вставить текст из kill-ring в позицию курсора

**ВАЖНО** — везде, где написано «удаляет», по факту текст вырезается (сохраняется в kill-ring и может быть вставлен через CTRL+Y).



### Дополнение (completion) — TAB

**TAB** дополняет ввод, если есть однозначное совпадение. Если совпадений несколько — звуковой сигнал (нажать TAB второй раз = список вариантов).

Работает не только с путями, но и с:
- переменными (`$...`)
- именами пользователей (`~...`)
- командами (первое слово в строке)
- сетевыми именами из `/etc/hosts` (`@...`)

**Клавиши:**
- __ALT+?__ — показать список дополнений (аналог второго TAB)
- __ALT+*__ — вставить все варианты дополнения сразу

Есть ещё программируемое дополнение (правила под конкретные команды/приложения) — посмотреть можно через `set | less`.


### История команд (history)

Все команды хранятся в `~/.bash_history`.

**Посмотреть историю:**
```bash
history | less
history | grep /usr/bin    # найти команды с конкретным текстом
```

**Поиск в реальном времени — CTRL+R**

Нажать **CTRL+R**, начать вводить текст — bash сам находит совпадение из истории.
- **ENTER** — выполнить найденную команду
- **CTRL+J** — просто вставить её в строку (для правки перед запуском)
- **CTRL+R** ещё раз — искать следующее совпадение
- **CTRL+G** / **CTRL+C** — отменить поиск

**Подстановка по номеру записи:** `!88` — выполнить команду №88 из `history`.

**Остальные клавиши (редко используются):**
- __CTRL+P__ / __CTRL+N__ — пред./след. запись (как ↑ / ↓)
- __ALT+< __ / __ALT+>__ — в начало / конец истории
- __ALT+P__ / __ALT+N__ — поиск назад/вперёд без инкремента
- __CTRL+O__ — выполнить текущую команду и перейти к следующей

### Владельцы, группы, остальные (permissions basics)

В Unix у файла есть **владелец** (user) и **группа**. Владелец управляет доступом; членам группы можно дать отдельные права; для всех остальных (**world**) — свои права.

**Команда `id`** — показывает вашу идентичность:
```bash
id
# uid=1000(me) gid=1000(me) groups=...
```
- __uid__ — числовой идентификатор пользователя
- __gid__ — идентификатор основной группы
- __groups__ — все группы, в которые входит пользователь

**Где хранится:**
- `/etc/passwd` — учётные записи (имя, uid, gid, домашний каталог, login shell)
- `/etc/group` — информация о группах
- `/etc/shadow` — пароли (обычным пользователям недоступен для чтения — отсюда ошибки `Permission denied`)

В Fedora обычные пользователи начинаются с uid 500, в Ubuntu — с 1000.
В современных дистрибутивах каждому пользователю обычно создают свою уникальную группу (с тем же именем), а не общую `users`.



### Права доступа: чтение, запись, выполнение (rwx)

Вывод `ls -l` показывает 10 символов атрибутов, например: `-rw-rw-r--`

**Первый символ — тип файла:**
- __-__ — обычный файл
- __d__ — каталог
- __l__ — символическая ссылка (атрибуты всегда `rwxrwxrwx`, реальные права — у файла, на который она ссылается)
- __c__ — символьное устройство (терминал, модем)
- __b__ — блочное устройство (жёсткий диск, CD-ROM)

**Остальные 9 символов** — права для владельца / группы / всех остальных (по 3 символа `rwx` на каждого).

**Что значит r/w/x для файлов:**
- __r__ — открывать и читать содержимое
- __w__ — записывать/усекать файл (но не переименовывать и не удалять — это определяется правами каталога)
- __x__ — выполнять как программу (скрипты дополнительно должны быть доступны для чтения)

**Что значит r/w/x для каталогов:**
- __r__ — читать содержимое каталога (только вместе с __x__)
- __w__ — создавать, удалять, переименовывать файлы внутри (только вместе с __x__)
- __x__ — входить в каталог (`cd`)

**Примеры атрибутов:**

- `-rwx------` — обычный файл, владелец: чтение/запись/выполнение, остальным — ничего
- `-rw-------` — обычный файл, владелец: чтение/запись, остальным — ничего
- `-rw-r--r--` — владелец: чтение/запись; группа и все остальные: только чтение
- `-rwxr-xr-x` — владелец: чтение/запись/выполнение; группа и все остальные: чтение/выполнение
- `-rw-rw----` — владелец и группа: чтение/запись; остальные — ничего
- `lrwxrwxrwx` — символическая ссылка (атрибуты недействительны, реальные права — у целевого файла)
- `drwxrwx---` — каталог: владелец и группа могут входить, создавать/переименовывать/удалять файлы внутри
- `drwxr-x---` — каталог: владелец может входить и управлять файлами внутри; группа может только входить, но не может создавать/переименовывать/удалять файлы



### chmod — изменение прав доступа

Менять права может только **владелец** или **суперпользователь**. Два способа: восьмеричный и символический.

**Восьмеричные числа (каждая цифра = 3 бита = rwx):**
- __0__ = `---` (000)
- __1__ = `--x` (001)
- __2__ = `-w-` (010)
- __3__ = `-wx` (011)
- __4__ = `r--` (100)
- __5__ = `r-x` (101)
- __6__ = `rw-` (110)
- __7__ = `rwx` (111)

Три цифры подряд = владелец / группа / остальные, например `chmod 600 file` → `rw-------`

Восьмеричная и шестнадцатеричная системы удобны для человека, т.к. компактно сворачивают группы битов (восьмеричная — по 3 бита, шестнадцатеричная — по 4 бита).

**Символическая форма — "для кого" (u/g/o/a):**
- __u__ — user, владелец
- __g__ — group, группа
- __o__ — others, остальные
- __a__ — all (u+g+o вместе); если не указано ничего — подразумевается `a`

**Операции:** `+` добавить, `-` отнять, `=` установить только эти права (остальные снять)

**Примеры символической формы:**
- `u+x` — добавить выполнение только владельцу
- `u-x` — отнять выполнение у владельца
- `+x` — добавить выполнение всем (владелец+группа+остальные), эквивалент `a+x`
- `o-rw` — отнять чтение и запись у всех, кроме владельца и группы
- `go=rw` — установить чтение+запись группе и остальным (снимет выполнение, если было)
- `u+x,go=rx` — добавить выполнение владельцу И установить чтение+выполнение группе/остальным (через запятую — несколько операций разом)

Флаг `--recursive` применяется и к файлам, и к каталогам одинаково — часто не то, что нужно, т.к. для них обычно нужны разные права.


### umask — права по умолчанию для новых файлов

`umask` не задаёт права напрямую (как `chmod`), а определяет маску — **что вычесть** из базовых прав при создании нового файла/каталога.

**Базовые права (фиксированные, не зависят от umask):**
- Файлы: `666` (`rw-rw-rw-`)
- Каталоги: `777` (`rwxrwxrwx`)

**Как работает маска:**
Там, где в маске (в двоичном виде) стоит **1** — соответствующий бит **вычитается** из базовых прав. Где **0** — бит остаётся как есть.

**Важно:** цифры в `umask` означают **не то же самое**, что в `chmod`! `chmod 022` **устанавливает** права `-w--w-`. А `umask 022` **вычитает** право записи у группы и остальных из базовых `666`/`777` — то есть это разные логики за одинаковыми на вид цифрами.

### Специальные биты прав: setuid, setgid, sticky

Права можно писать 4 цифрами: первая (крайняя левая) — особые биты.

**setuid (4000)** — на исполняемом файле: пока программа работает, процесс получает права **владельца файла**, а не запустившего пользователя.
- Пример: `/usr/bin/passwd` принадлежит root и имеет setuid → обычный пользователь запускает `passwd` **без sudo**, программа сама на время работы получает права root, чтобы записать новый пароль в `/etc/shadow` (куда обычному юзеру писать нельзя).
- Отличие от `sudo`: `sudo` даёt пользователю root вообще для любой команды (и требует быть в списке sudoers). setuid даёт root-права только этой одной программе, только для того, что она умеет делать — гораздо более узкое и безопасное разрешение.
- Пароль root при этом **не спрашивается** — программа просто проверяет ваш собственный пароль.
- На каталоге — не действует (игнорируется).
- Установить: `chmod u+s program`

**setgid (2000)** — аналогично setuid, но для группы.
- На файле-программе: процесс получает права группы-владельца файла.
- На каталоге (частый полезный случай): все новые файлы, создаваемые внутри, автоматически получают **группу папки**, а не группу создателя — удобно для общих папок команды.
- Установить: `chmod g+s dir`

**sticky bit (1000)** — только для каталогов (на файлах игнорируется).
- Даже если папка открыта на запись всем, **удалить файл может только**: его владелец, владелец папки, или root — остальные не могут удалить чужие файлы.
- Классический пример — `/tmp`.
- Установить: `chmod +t dir`

**Как выглядит в `ls -l`:**
- `-rwsr-xr-x` — `s` вместо `x` у владельца → setuid
- `drwxrwsr-x` — `s` вместо `x` у группы → setgid
- `drwxrwxrwt` — `t` вместо `x` у остальных → sticky bit

Установить эти биты на **свой** файл/папку можно и обычному пользователю, но польза от setuid/setgid появляется только если владелец файла — привилегированный (например root); иначе бит просто ничего не меняет.




### su — запуск оболочки от имени другого пользователя

Синтаксис: `su [-[l]] [пользователь]`

**Без аргументов** — переключает на root, спрашивает **пароль root**:
```bash
su
Пароль:
# ← промпт с "#" вместо "$" = права root
```
`exit` — вернуться обратно к своему пользователю.

**Флаг `-l` (или сокращённо просто `-`)** — запускает оболочку **входа**: загружает окружение указанного пользователя и переходит в его домашний каталог. Обычно это то, что нужно.
```bash
su -l someuser
su - someuser    # то же самое, сокращённая форма
```
Без пользователя после `-l` / `-` — подразумевается root (переход в `/root`).

**Выполнить одну команду, не открывая интерактивную сессию:**
```bash
su -c 'команда'
```
Команду обязательно в кавычках — иначе подстановки (`$`, `*` и т.д.) обработает **текущая** оболочка до передачи в su, а не оболочка root.

Пример:
```bash
su -c 'ls -l /root/*'
Пароль:
```


### sudo — выполнение команды от имени другого пользователя

Похож на `su`, но с важными отличиями:

- **Пароль вводится СВОЙ, а не root'а** (в отличие от `su`)
- Администратор может ограничить, **какие** команды разрешено запускать конкретному пользователю через sudo (не обязательно все)
- **Не открывает новую оболочку** и не грузит окружение другого пользователя — команды выполняются как обычно, без дополнительного экранирования
- Можно запустить интерактивную root-сессию: `sudo -i` (похоже на `su -`)

**Пример:**
```bash
sudo backup_script
Пароль:   ← ваш собственный пароль
```

**Посмотреть свои привилегии в sudo:**
```bash
sudo -l
# User warcry may run the following commands on wlmd91:
#    (ALL : ALL) ALL
```

**Почему в Ubuntu именно sudo, а не su:**
- В Ubuntu **нет пароля** у учётной записи root — войти как root через `su` невозможно
- Вместо этого первому пользователю сразу выдаются полные права через `sudo`, и он может раздавать их другим
- Идея: не давать root-доступ "навсегда за один вход" (как в Windows-стиле с админ-правами), а выдавать привилегии **точечно**, по одной команде — это безопаснее


### chown — изменение владельца и группы

Синтаксис: `chown [владелец][:[группа]] файл...`

Требует прав суперпользователя (sudo).

**Примеры аргументов:**
- `bob` — назначить владельцем `bob`
- `bob:users` — назначить владельцем `bob`, группой `users`
- `:admins` — сменить только группу на `admins`
- `bob:` — назначить владельцем `bob` и его основную группу

**Пример из книги:** `janet` (с sudo) копирует файл в домашний каталог `tony`:
```bash
sudo cp myfile.txt ~tony
# [sudo: authenticate] Password:

sudo ls -l ~tony/myfile.txt
# -rw-r--r-- 1 root root 8031 2018-03-20 14:30 /home/tony/myfile.txt

sudo chown tony: ~tony/myfile.txt

sudo ls -l ~tony/myfile.txt
# -rw-r--r-- 1 tony tony 8031 2018-03-20 14:30 /home/tony/myfile.txt

```
После копирования файл принадлежит `root` (из-за sudo) — `chown tony:` меняет владельца и группу на `tony`.

После первого ввода пароля `sudo` обычно **не спрашивает пароль повторно** несколько минут — "доверие" действует, пока не истечёт таймаут.



### chgrp — изменение группы файла

В старых версиях Unix `chown` мог менять только владельца, но не группу — для смены группы использовалась отдельная команда `chgrp`.

Работает похоже на `chown`, но имеет больше ограничений.



### passwd — смена пароля

Синтаксис: `passwd [пользователь]`

**Свой пароль:**
```bash
passwd
```
Спросит текущий пароль, затем новый.

`passwd` проверяет пароль на "силу" — отклоняет слишком короткие, похожие на старый, или словарные пароли.

**Пароль другого пользователя** (нужны привилегии суперпользователя):
```bash
sudo passwd имя_пользователя
```

Суперпользователю также доступны блокировка учётных записей, срок действия пароля и др. (см. `man passwd`).



## Процессы

Linux — многозадачная система: ядро создаёт иллюзию одновременности, быстро переключаясь между программами через **процессы**.

**Команды главы:**
- `ps` — список процессов
- `top` — задачи (в реальном времени)
- `jobs` — активные задания
- `bg` — перевести задание в фон
- `fg` — перевести задание на передний план
- `kill` — послать сигнал процессу
- `killall` — остановить процессы по имени
- `shutdown` — остановить/перезагрузить систему

### Как работает процесс

- При старте системы ядро запускает `init` — первый процесс, всегда с **PID 1**
- `init` запускает сценарии начальной загрузки (из `/etc`), которые поднимают системные службы
- Многие службы — это **демоны** (daemon): работают в фоне, без участия пользователя
- Принцип запуска: **родительский процесс** запускает **дочерний процесс**
- Каждому процессу присваивается **PID** (Process ID), по возрастанию
- Ядро отслеживает у процесса: память, готовность к выполнению, владельца/группу (как у файлов), эффективный (действующий) ID пользователя


### ps — просмотр списка процессов

**Простой вызов** — только процессы текущего сеанса:
```bash
ps
# PID TTY     TIME     CMD
# 402 pts/0   00:00:00 zsh
# 608 pts/0   00:00:00 ps
```
- __PID__ — идентификатор процесса
- __TTY__ — управляющий терминал (тип терминала)
- __TIME__ — потреблённое процессорное время
- __CMD__ — команда/программа

**`ps x`** — все процессы, включая те, что без управляющего терминала (у них в TTY стоит `?`).

Добавляется столбец **STAT** (состояние процесса):
- __R__ — выполняется / готов к выполнению
- __S__ — приостановлен (ждёт события — клавишу, сеть и т.п.)
- __D__ — приостановлен без возможности прерывания (ждёт I/O, например диск)
- __T__ — остановлен принудительно
- __Z__ — зомби (дочерний процесс завершился, но родитель его не удалил)
- __<__ — высокий приоритет (менее "уступчив", забирает больше CPU)
- __N__ — низкий приоритет (более "уступчив", получает CPU после остальных)

(могут быть доп. символы после статуса — см. `man ps`)

**`ps aux`** — процессы **всех** пользователей, в "стиле BSD" (без дефиса перед параметрами). Добавляет столбцы:
- __USER__ — владелец процесса
- __%CPU__ — использование процессора, %
- __%MEM__ — использование памяти, %
- __VSZ__ — объём виртуальной памяти
- __RSS__ — физическая память (ОЗУ) в кб
- __START__ — время запуска (дата, если больше 24 часов назад)

Т.к. список длинный — удобно смотреть через `ps aux | less`.


### top — процессы в динамике

В отличие от `ps` (снимок на момент вызова), `top` **постоянно обновляет** информацию (по умолчанию каждые 3 сек).

```bash
top
```

**Сводная часть (верх экрана):**
- __время + uptime__ — сколько система работает с последней загрузки
- __users__ — сколько пользователей в системе
- __load average__ — среднее число процессов, ожидающих CPU; 3 значения = за 1 / 5 / 15 минут. Ниже 1.0 = система не нагружена
- __Tasks__ — сколько процессов всего и в каких состояниях
- __%us__ — CPU на пользовательские процессы
- __%sy__ — CPU на системные процессы (ядро)
- __%ni__ — CPU на низкоприоритетные (nice) процессы
- __%id__ — простой CPU
- __Mem__ — использование физической памяти (ОЗУ)
- __Swap__ — использование файла подкачки

**Ниже** — таблица процессов, отсортированная по потреблению CPU.

**Команды внутри top:**
- __h__ — справка
- __q__ — выход


### Управление процессами: прерывание и фоновый режим

**Обычный запуск** (например `gedit`) — оболочка блокируется, ждёт завершения программы. Пока окно открыто — новый промпт не появляется. Закрыли окно — промпт вернулся.

**Прерывание процесса — CTRL+C:**
```bash
gedit
```
Нажать **CTRL+C** в терминале → программа закрывается, оболочка возвращает приглашение к вводу. Работает для многих (но не всех) программ командной строки.

**Перевод в фоновый режим — `&`:**
```bash
gedit &
[1] 28236
```
Программа запускается, окно появляется, но оболочка **сразу** возвращает приглашение — не дожидаясь закрытия программы.
- `[1]` — номер задания (job)
- `28236` — PID процесса

**Посмотреть активные задания в терминале:**
```bash
jobs
[1]+ Running    gedit &
```

**Убедиться, что процесс работает через `ps`:**
```bash
ps
PID TTY  TIME CMD
... gedit
```

### Возврат процесса на передний план — fg

Фоновый процесс **не получает** ввод с клавиатуры (в т.ч. CTRL+C не остановит его в фоне).

Чтобы вернуть на передний план:
```bash
fg %1
```
`%1` — спецификатор задания (jobspec): знак процента + номер из `jobs`. Если фоновое задание одно — номер можно опустить, просто `fg`.

После возврата на передний план программу снова можно прервать через **CTRL+C**.


### Приостановка процесса — CTRL+Z, возврат на передний план — fg, перевести в фоновый режим — bg

**CTRL+Z** — приостанавливает процесс переднего плана (не завершает, просто "замораживает"):
```bash
gedit
[1]+ Stopped    gedit
```
Пока приостановлен — программа никак не реагирует (например, окно не перерисовывается при изменении размера).

**Вернуть на передний план:**
```bash
fg %1
```
**Или перевести в фоновый режим (продолжит работать в фоне):**
```bash
bg %1
[1]+ gedit &
```

Спецификатор `%1` можно опустить, если задание всего одно.

**Полезно, если забыли добавить `&`** при запуске GUI-программы из терминала — можно приостановить (CTRL+Z) и потом перевести в фон (`bg`), не перезапуская программу заново.

**Зачем вообще запускать GUI-программы из терминала:**
- программы может не быть в меню приложений
- в терминале видны сообщения об ошибках (полезно, если программа падает при запуске из графического меню)
- у некоторых GUI-программ есть полезные параметры командной строки


### kill — сигналы процессам

`kill` не просто "убивает" процесс — она **посылает ему сигнал**. Сигналы — способ общения ОС с программами.

**Использование:**
```bash
xlogo &
[1] 28401
kill 28401      # по PID
kill %1         # или по спецификатору задания
```

**Знакомые сигналы из комбинаций клавиш:**
- **CTRL+C** → сигнал **INT** (Interrupt)
- **CTRL+Z** → сигнал **TSTP** (Terminal Stop)

Программа сама решает, как реагировать на сигнал (например, сохранить данные перед завершением).

---

**Важный нюанс (проверено на практике):**

Если процесс **приостановлен** (CTRL+Z, состояние Stopped) и попробовать `kill PID` — сигнал **встаёт в очередь, но не обрабатывается**, потому что процесс не выполняется и не может отреагировать. Он "оживёт" и закроется только после `fg` или `bg`.

**Но `kill %1`** (по спецификатору задания, а не по PID) — срабатывает **мгновенно**, даже если процесс приостановлен. Это особенность самой оболочки (zsh/bash): она сама "будит" задание перед отправкой сигнала.

**Всегда мгновенно, независимо от состояния процесса:**
```bash
kill -9 PID   # SIGKILL — обрабатывается ядром напрямую, минуя саму программу
```


### kill — конкретные сигналы

Синтаксис: `kill [-сигнал] PID...`
Если сигнал не указан — по умолчанию посылается **TERM**.

Сигнал можно указывать по номеру, по имени (`-INT`) или с префиксом `SIG` (`-SIGINT`).

**Частые сигналы:**
- __1 / HUP__ — обрыв связи (изначально — потеря связи с терминалом). Многие демоны используют его как команду "перечитать конфиг и перезапуститься" (например Apache)
- __2 / INT__ — прервать, аналог CTRL+C, обычно завершает программу
- __9 / KILL__ — уничтожить немедленно. **Не передаётся программе** — ядро завершает процесс само, минуя саму программу, без шанса "прибраться" или сохранить данные. Использовать только как крайнюю меру
- __15 / TERM__ — завершить (сигнал по умолчанию для `kill`). Если программа "жива" — завершится корректно
- __18 / CONT__ — продолжить после STOP
- __19 / STOP__ — приостановить. Как и KILL, **не передаётся** программе — нельзя игнорировать
- __20 / TSTP__ — сигнал "стоп" с клавиатуры (аналог CTRL+Z). В отличие от STOP — передаётся программе, и она может его проигнорировать

**Другие сигналы:**
- __3 / QUIT__ — выйти
- __11 / SEGV__ — ошибка сегментации (недопустимое обращение к памяти)
- __28 / WINCH__ — изменение размера окна терминала (реагируют, например, `top` и `less`)

**Примеры:**
```bash
kill -1 13546        # HUP
kill -INT 13601      # INT по имени
kill -SIGINT 13608   # INT с префиксом SIG
```

Чтобы послать сигнал — нужно быть **владельцем процесса** (или суперпользователем).
Полный список сигналов: `kill -l`


### killall — сигнал сразу нескольким процессам по имени

Синтаксис: `killall [-u пользователь] [-сигнал] имя...`

В отличие от `kill` (работает по PID/jobspec), `killall` находит и завершает процессы **по имени программы** — удобно, если несколько экземпляров запущено разом.

**Пример:**
```bash
xlogo &
[1] 18801
xlogo &
[2] 18802

killall xlogo
[1]- Terminated    xlogo
[2]+ Terminated    xlogo
```
Оба экземпляра завершились одной командой.

Как и с `kill` — нужно быть владельцем процесса (или суперпользователем), чтобы посылать сигнал.


### Остановка системы: halt, poweroff, reboot, shutdown

Четыре команды (нужны права суперпользователя, `sudo`):

- __halt__ — остановить систему
- __poweroff__ — выключить электропитание
- __reboot__ — перезагрузить
```bash
sudo reboot
```

**shutdown** — более гибкая: позволяет указать действие и задержку перед ним.

```bash
sudo shutdown -h now    # остановить (halt)
sudo shutdown -r now    # перезагрузить (reboot)
```

Разные варианты задержки — см. `man shutdown`.
После запуска `shutdown` рассылает предупреждение всем зарегистрированным в системе пользователям о предстоящем событии.


### Другие команды мониторинга процессов

- __pstree__ — процессы в виде дерева (отношения "родитель–потомок")
- __vmstat__ — снимок использования ресурсов (память, файл подкачки, дисковый I/O). С интервалом (в секундах): `vmstat 5` — обновляется, пока не нажмёте CTRL+C
- __xload__ — GUI-график нагрузки на систему во времени
- __tload__ — то же самое, но график рисуется прямо в терминале; завершается CTRL+C

Понравилась `pstree` — наглядно видна иерархия процессов.
